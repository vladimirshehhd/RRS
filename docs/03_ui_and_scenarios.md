# Этап 3. UX-анализ: интерфейс, сценарии и ошибки

**Файл:** `docs/03_ui_and_scenarios.md`

## 1. Список экранов / страниц
Минимальный набор:
- **Главная страница**
- **Страница настроек**
- **Страница “О проекте”**

## 2. Элементы на каждом экране

### Главная страница
- Заголовок сайта.
- Выпадающий список источников RSS.
- Кнопка **«Обновить»**.
- Поле поиска / фильтрации по тексту.
- Блок “Последнее обновление”.
- Список карточек новостей.
- У каждой карточки: заголовок, дата, источник, краткое описание, кнопка/ссылка **«Читать полностью»**.

### Страница настроек
- Поле выбора количества новостей на странице.
- Переключатель “использовать кэш при ошибке сети”.
- Поле таймаута запроса (только в учебных целях, можно ограничить списком значений).
- Кнопка **«Сохранить»**.

### Страница “О проекте”
- Краткое описание проекта.
- Используемые технологии.
- Список ограничений MVP.

## 3. Состояния UI
- **Empty:** источник выбран, но новостей нет; показываем текст “Новостей пока нет”.
- **Loading:** во время обновления показываем индикатор и текст “Загружаем RSS…”.
- **Success:** отображается список новостей.
- **Error:** показываем баннер/сообщение с понятным описанием ошибки.

## 4. Пользовательские сценарии

### Сценарий 1. Просмотр свежих новостей
1. Пользователь открывает сайт.
2. Главная страница автоматически загружает ленту по умолчанию.
3. Пользователь видит список последних новостей.
4. Пользователь нажимает “Читать полностью” у интересующей записи.

### Сценарий 2. Переключение источника
1. Пользователь открывает выпадающий список источников.
2. Выбирает другой RSS‑источник.
3. Сайт перезагружает ленту.
4. На экране появляется новый список новостей для выбранного источника.

### Сценарий 3. Поиск по заголовкам
1. Пользователь вводит слово в поле поиска.
2. Сайт фильтрует уже загруженные новости.
3. Пользователь видит только записи, где есть совпадение в заголовке или описании.
4. При очистке поля снова показывается полный список.
