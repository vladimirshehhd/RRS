# Этап 7. Конфигурация, хранение данных, ограничения и политика ошибок

**Файл:** `docs/07_config_storage_constraints.md`

## 1. Конфиг-параметры и значения по умолчанию
`default_source_id = "weird-news"`
`items_per_page = 20`
`cache_ttl_minutes = 15`
`network_timeout_seconds = 5`
`use_cache_on_error = true`
`max_sources = 5`

## 2. Где храним локальные файлы / данные

Windows: %APPDATA%/RssNews/
Linux: ~/.config/RssNews/

## 3. Политика кэша
- Кэшируем результат загрузки RSS по каждому источнику.
- TTL 10 минут

## 4. Ограничения / лимиты
API: Бесплатный API (100 запросов в минуту).
Платформа: Не более 5 запросов в секунду к одному хосту.

## 5. Политика ошибок

### Что показываем пользователю
- Короткое понятное сообщение без технического мусора.
- При наличии кэша показываем предупреждение, например: **“Показана последняя сохранённая версия ленты.”**

### Что логируем
- Время ошибки.
- Тип ошибки (`NetworkError`, `InvalidFeedFormatError`, `StorageError`).
- URL RSS‑источника.
- HTTP‑статус или текст исключения.
- Краткий stack trace только для режима разработки.
