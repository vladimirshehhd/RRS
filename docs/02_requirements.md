# Этап 2. Требования (MVP)

**Файл:** `docs/02_requirements.md`

## MUST — обязательные требования

### MUST-1. Загрузка новостей из RSS
Сайт должен получать данные хотя бы из одного внешнего RSS‑источника.

**Критерий приёмки:** при открытии главной страницы отображается не менее 5 новостей из настроенного RSS‑источника, если источник доступен.

### MUST-2. Список новостей на главной странице
Сайт должен показывать список новостей в виде карточек или строк.

**Критерий приёмки:** для каждой новости на главной странице видны заголовок, дата публикации, источник и краткое описание.

### MUST-3. Переход к полной новости
Пользователь должен иметь возможность перейти к полной публикации на внешнем сайте.

**Критерий приёмки:** у каждой новости есть ссылка “Читать полностью”, по которой открывается оригинальная статья.

### MUST-4. Обновление ленты
Пользователь должен иметь возможность обновить данные вручную.

**Критерий приёмки:** после нажатия кнопки “Обновить” приложение выполняет повторное чтение RSS и показывает более свежие записи, если они появились.

### MUST-5. Фильтрация новостей
Пользователь должен иметь возможность отфильтровать видимые новости по текстовому запросу.

**Критерий приёмки:** при вводе слова в поле поиска список новостей сокращается и остаются только записи, где слово встречается в заголовке или описании.

### MUST-6. Выбор источника
Пользователь должен иметь возможность выбрать один из заранее заданных RSS‑источников.

**Критерий приёмки:** при переключении источника список новостей обновляется и показывает записи только из выбранной ленты.

### MUST-7. Устойчивость к пустой/ошибочной ленте
Приложение должно корректно обрабатывать ошибки сети, пустые ответы и некорректный RSS.

**Критерий приёмки:** при недоступном источнике пользователь видит понятное сообщение об ошибке, а сервер не завершается аварийно.

### MUST-8. Защита от дубликатов
Приложение не должно показывать одну и ту же новость несколько раз в пределах одного источника.

**Критерий приёмки:** если RSS содержит повторяющиеся записи с одинаковой ссылкой или GUID, в интерфейсе показывается только одна запись.

## NICE — желательные требования
- Сохранение последнего выбранного источника между перезапусками.
- Отображение времени последнего успешного обновления.
- Сортировка новостей по дате публикации (сначала новые).
- Страница “О проекте” с кратким описанием сайта.
- Простая тёмная тема через CSS.
